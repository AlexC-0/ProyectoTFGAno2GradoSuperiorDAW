1. OBJETIVO DEL DOCUMENTO


====================================================================


Este documento describe el prototipado funcional y visual del TFG GR-Inn / ECO & WOODS en su estado actual. Su finalidad es dejar constancia de como debe representarse la aplicacion en una maqueta, wireframe, documento de memoria o exposicion del proyecto.

A diferencia de una explicacion puramente textual, este documento incluye dibujos en texto plano mediante lineas, cajas, separadores y botones simulados. Estos wireframes no son imagenes externas, sino esquemas ASCII integrados dentro del propio documento para representar la forma aproximada de cada pantalla.

El documento no plantea una version futura ni idealizada. Representa la aplicacion existente: una plataforma web de compraventa de muebles de segunda mano y recambios 3D, desarrollada con PHP, MySQL, sesiones de usuario, carrito, favoritos, resenas, mensajeria y panel de administracion.


====================================================================

2. IDENTIDAD GENERAL DEL PRODUCTO


====================================================================


Nombre visible del proyecto:


- GR-Inn en la pagina de inicio y varias pantallas.

- ECO & WOODS en titulos internos y textos historicos del proyecto.


Enfoque del producto:


- Mercado circular para muebles.

- Compra y venta de muebles usados.

- Reutilizacion y consumo responsable.

- Reparacion mediante recambios 3D.

- Contacto directo entre usuarios.

- Gestion personal de anuncios, favoritos, carrito, mensajes y resenas.


Idea que debe transmitir el prototipo:


La aplicacion debe verse como una plataforma sencilla, clara y funcional, centrada en alargar la vida util de los muebles mediante segunda mano y piezas de reparacion.


====================================================================

3. USUARIOS CONTEMPLADOS


====================================================================


3.1. Visitante no autenticado


Puede ver el inicio, acceder al login, registrarse y explorar contenido publico. Las acciones privadas le llevan al inicio de sesion.

3.2. Usuario registrado


Puede explorar catalogos, guardar favoritos, anadir productos al carrito, publicar muebles, escribir resenas, contactar con vendedores y gestionar su perfil.

3.3. Administrador


Puede realizar las acciones de usuario registrado y, ademas, gestionar usuarios, muebles, recambios 3D y resenas desde el panel de administracion. Tambien puede publicar recambios 3D.


====================================================================

4. ESTILO VISUAL ACTUAL


====================================================================


El prototipo debe respetar la identidad visual actual del proyecto.

Paleta principal:


- Fondo general claro, tipo beige o crema suave.

- Verde como color principal de marca.

- Tonos madera o marron como apoyo visual.

- Paneles oscuros en verde profundo para bloques destacados.

- Blanco para formularios, tarjetas y tablas.


Elementos visuales recurrentes:


- Cabecera superior con logo.

- Navegacion horizontal con enlaces redondeados.

- Tarjetas para productos, favoritos, resenas y mensajes.

- Botones redondeados.

- Formularios con campos amplios y bordes suaves.

- Etiquetas o badges para categoria, ubicacion, tipo o compatibilidad.

- Tablas para carrito, resumen de compra y administracion.

- Mensajes de exito y error.

- Boton flotante para volver arriba.

- Toasts o avisos breves para carrito, favoritos y resenas.


Tipografia:


- Titulares con aspecto elegante.

- Texto general legible.

- Jerarquia clara entre titulo, subtitulo, informacion y acciones.

6. LEYENDA DE WIREFRAMES


====================================================================


+-----+        Caja, panel, contenedor o tarjeta.
|     |        Contenido dentro de un bloque.

[     ]        Boton, enlace o campo de formulario.
(  )          Icono, contador o elemento pequeno.
.....         Texto descriptivo o contenido variable.
====          Separador de seccion importante.
----          Separador secundario.
< >           Selector o desplegable.
{ }           Grupo de tarjetas o resultados.

Los wireframes muestran distribucion, no diseno final exacto. Sirven para explicar la estructura de la interfaz en memoria o defensa.

11. ADAPTACION RESPONSIVE


====================================================================


El prototipo debe prever que la interfaz se adapte a distintos tamanos de pantalla.

11.1. Escritorio


- Cabecera horizontal.

- Tarjetas en grid de hasta tres columnas.

- Formularios de dos columnas cuando tenga sentido.

- Tablas completas.


11.2. Tablet


- Menor numero de columnas.

- Formularios mas compactos.

- Tarjetas en una o dos columnas.


11.3. Movil


- Navegacion reorganizada o desplazable.

- Tarjetas en una sola columna.

- Formularios en una sola columna.

- Botones ocupando mas ancho.

- Tablas con scroll horizontal.

12. ALCANCE ACTUAL Y LIMITES DEL PROTOTIPO


====================================================================


El prototipo debe incluir:


- Inicio.

- Login y registro.

- Catalogo de muebles.

- Detalle de mueble.

- Catalogo de recambios 3D.

- Detalle de recambio.

- Publicacion de muebles.

- Publicacion de recambios para administrador.

- Mi perfil.

- Edicion de muebles.

- Mensajeria.

- Carrito.

- Finalizacion de compra en modo prueba.

- Panel de administracion.

- Edicion de recambios.


El prototipo no debe presentar como funcionalidad real:


- Pago online real.

- Direcciones de envio.

- Sistema de valoracion de vendedores independiente.

- Chat en tiempo real.

- Edicion de imagenes ya subidas.

- Publicacion de recambios por usuarios normales.

- Panel avanzado de estadisticas.


====================================================================

13. CRITERIO DE FIDELIDAD


====================================================================


Un prototipo fiel al estado actual del TFG debe representar la aplicacion como una plataforma ya navegable, con flujos completos pero sencillos:

- El visitante puede explorar y registrarse.

- El usuario registrado puede comprar en modo prueba, guardar favoritos, publicar muebles, escribir resenas, contactar vendedores y gestionar su perfil.

- El administrador puede gestionar usuarios, muebles, recambios y resenas, ademas de publicar recambios 3D.


La maqueta debe mantener el caracter del proyecto: reutilizacion de muebles, reparacion mediante piezas 3D, sencillez de uso y gestion responsable de publicaciones.
